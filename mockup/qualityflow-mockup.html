1. Overview

This document defines the technical specification for the API Gateway service responsible for routing, authentication, and rate limiting across all microservices in the platform.

2. Authentication

All incoming requests must carry a valid JWT token issued by the Internal Auth service. Token validation occurs at the gateway layer before forwarding to downstream services.

Token expiry is set to 3600 seconds with a sliding window refresh mechanism enabled by default.

Tokens that fail validation will receive a 401 response with a standardized error payload. No forwarding occurs for invalid tokens under any circumstances.

💬 "Token expiry is set to 3600 seconds"

3. Rate Limiting

Rate limits are enforced per client ID using a token-bucket algorithm. Default limits are configured at the gateway level and can be overridden per route.

Burst capacity is fixed at 500 requests/minute with no configurable override at runtime.

All limit breach events are logged to the central observability stack with severity INFO. No automatic alerting is currently configured for repeated violations.

4. Routing Rules

Routes are defined in a YAML configuration file loaded at startup. Dynamic route registration via API is planned for Q3 but not in scope for this release.

Token rotation frequency defaults to every 24h for service accounts.

5. Error Handling

All errors are returned as JSON with a standardized error_code and message field. HTTP status codes follow RFC 7231 conventions.
